# Philly Attractions
 
**A multi-page tourism website showcasing the best entertainment, food, and shopping destinations Philadelphia has to offer.**
 
Built between September 2025 and December 2025.
 
---
 
## Overview
 
Philly Attractions is a web application designed to make it easy for tourists and locals alike to discover what Philadelphia has to offer. Rather than sifting through scattered blog posts and outdated travel guides, users can browse a clean, organized, multi-page site that highlights top entertainment venues, restaurants, and shopping destinations across the city, complete with practical details like location, hours, and visual previews.
 
## The Problem
 
Exploring a new city, or rediscovering your own, often comes with friction:
 
- **Scattered information** - attraction details are spread across dozens of sites with inconsistent formatting.
- **Missing practical details** - many guides skip operating hours, exact locations, or accessibility information.
- **Poor visual context** - text-only listings make it hard to know what to expect before visiting.
- **No streamlined navigation** - users are forced to jump between maps, review sites, and hours pages separately.
Philly Attractions consolidates all of this into a single, easy-to-navigate site built specifically around Philadelphia's entertainment, food, and shopping scene.
 
## Key Features
 
- **Multi-Page Site Structure** - Dedicated pages for entertainment, food, and shopping destinations, each organized for easy browsing.
- **Interactive UI Components** - Four custom interactive components built in JavaScript, giving the site a dynamic, app-like feel rather than static pages.
- **Responsive CSS Layouts** - Fully responsive design and visual styling that adapts cleanly across desktop and mobile screens.
- **Geolocation Details** - Each destination includes location information to help users navigate to attractions with ease.
- **Operating Hours** - Up-to-date hours listed for every destination, removing the guesswork of a wasted trip.
- **Visual Content Integration** - Photos and visual previews for each destination, giving users a sense of what to expect before they arrive.
## Tech Stack
 
| Layer | Technology |
|---|---|
| Structure | HTML |
| Styling | CSS |

## How It Works
 
1. **Browse** - Users land on the homepage and navigate to the category that interests them: entertainment, food, or shopping.
2. **Explore** - Each destination is presented with visual content, a short description, location details, and operating hours.
3. **Interact** - Custom JavaScript components let users filter, expand, or interact with listings rather than scrolling through static text.
4. **Navigate** - Geolocation details help users get from the site straight to the destination with minimal extra searching.
## Getting Started
 
### Prerequisites
- A modern web browser
- A local code editor (VS Code recommended) if you want to modify the project
- (Optional) A lightweight local server, such as the VS Code Live Server extension, for the best development experience
### Installation
 
```bash
# Clone the repository
git clone https://github.com/kamolaa7/PhillyProject.git
cd PhillyProject
 
# Open the project in your editor of choice
code .
 
# Open index.html directly in your browser, or serve it locally
# For example, using Live Server in VS Code, or:
npx serve .
```
